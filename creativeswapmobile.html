CreativeSwap Mobile

A cross-platform iOS and Android companion app for the CreativeSwap web platform. Built with React Native and Expo, it lets artists photograph their work and post trade listings directly from their phones.

The Problem

Artists work on the go - the web app wasn't enough

The CreativeSwap web platform works well at a desktop but artists are often at markets, studios, and events when they want to post a new listing. Pulling out a laptop to photograph and upload work isn't realistic in those situations.

The mobile companion app was built to solve exactly that - a fast, photo-first experience that lets artists post trade listings directly from their phone in seconds, with GPS-based distance filtering already built in.

The Solution

A photo-first mobile experience sharing the same Firebase backend

Rather than rebuilding a separate backend, the mobile app connects directly to the same Firebase project as the web platform - same authentication, same Firestore database, same Cloud Storage. This means listings created on mobile appear instantly on the web app and vice versa.

Photo-First Listings

Upload photos directly from the device gallery, preview and remove images before posting, and submit listings straight to Firebase Storage.

GPS Distance Filtering

Listings are filtered to a 50km radius using coordinates stored in the user's Firebase profile, with distance displayed on every listing card.

Saved Listings

Users can bookmark listings while browsing and access them later from a dedicated Saved screen. Bookmarks persist between sessions using AsyncStorage.

Full Listing Management

Create, edit, and delete listings from mobile. Edit screen pre-populates existing data and supports adding or removing individual photos.

Reflection

What I learned building for mobile

Mobile development introduced challenges that don't exist in web development - device permissions, keyboard behavior differences between iOS and Android, and image upload workflows that have to account for the device's file system rather than a browser.

The keyboard behavior on Android was a real challenge - form inputs were getting hidden behind the keyboard. The fix required both KeyboardAwareScrollView in the component and softwareKeyboardLayoutMode configured in app.json. Small config details like that can cost hours without knowing what to look for.

Sharing the Firebase backend with the web app was one of the best architectural decisions - it meant I could focus entirely on the mobile experience rather than rebuilding data logic that already existed. In a real product this would be the right approach for any companion app.
